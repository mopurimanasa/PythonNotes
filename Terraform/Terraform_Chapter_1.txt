main.tf --> In this we will define what type of resource that we want to create
With main.tf -> we create a file called providers.tf -> we define configuration of the cloud on which you want to create the resources -> for example
	if you want to create the resources of azure than we will define the azure configuration in providers.tf file
arm templates -> is another tool similar to provided by azure and it is only used for azure. arm templates are return in the JSON format. In the same way we have 
cloud formation for AWS.

we have commands in terraform:
->
terraform init: when we run command terraform init, it looks for the cloud service provider information in providers.tf. if we given as azure 
than it automatically download the packages of azure. so that's why provider.tf is very improtant.
terraform plan
terraform apply

variables.tf -> define inpputs for a particualr variables. 

-> After we keep the information in main.tf, providers.tf, variables.tf we can execute the terraform init.
-> After that we can execute the terraform plan -> starts executing the code, which we defined in the main.tf file. shows what all the resources that
we going to create. Modification, Deletion etc. 
-> After that we can execute terraform apply -> Than it will start creating the resources which mentioned in main.tf file into portal of mentioned provider.
