API Design and methods:

-> API design is the process of creating endpoints and rules that allow different systems to communicate with each other.
-> This follows a typical 3-tier architecture – presentation layer (frontend), business logic layer (backend), and data layer (database)
-> For example, in an e-commerce app, the React frontend sends a request when a user places an order, the backend processes it, and the database 
stores the order and updates inventory.

Common HTTP Methods:

1. GET (Read data)

-> Used to retrieve data
-> GET method is used to fetch data from the server without modifying it.

Example: 
GET /users
GET /users/101

2. POST (Create data)

-> Used to create new data
-> POST method is used to send data to the server to create a new resource.

Example:
POST /users

Body:
{
  "name": "Manasa",
  "role": "Engineer"
}

3. PUT (Update full data)

-> Used to update entire resource
-> PUT is used to completely update an existing resource.

Example:
PUT /users/101

Body:
{
  "id": 101,
  "name": "Manasa",
  "role": "Engineer"
}

4. PATCH (Update partial data)

-> Used to update specific fields
-> PATCH is used for partial updates of a resource.

Example:
PATCH /users/101

Body:
{
  "role": "Senior Engineer"
}

✅ What happens:
👉 The server:
Finds user with ID 101
Updates only the role field
Keeps other fields unchanged

5. DELETE (Delete data)

-> Used to remove data
-> DELETE method is used to remove a resource from the server.

Example: 
DELETE /users/101

->Usually no body is required

API Design Best Practices:

1. Use proper naming
/users
/orders
/products

2. Use nouns, not verbs
✅ /users
❌ /getUsers

3. Use status codes clearly
| Code | Meaning      |
| ---- | ------------ |
| 200  | Success      |
| 201  | Created      |
| 400  | Bad request  |
| 404  | Not found    |
| 500  | Server error |

4. Version your API
/api/v1/users

5. Keep responses consistent
{
  "status": "success",
  "data": {...}
}

--------------------------------------------------------------------------------------------------------------

-> In my project, I worked on designing REST APIs using Python.
-> We used GET for fetching data, POST for creating records, PUT/PATCH for updates, and DELETE for removal.
-> We also ensured proper status codes and followed RESTful naming conventions like /users and /transactions
